About

This website is an Application Programming Interface (API), which means it is a machine-readable resource, that delivers the publicly available content of directory.gov.au according to Semantic Web principles.

The goal of establishing this website is to demonstrate how machine-readable contents of directory.gov.au could look in the future and how Semantic Web and Linked Data systems may interact with it.

How the data is produced

The data for this API is taken from directory.gov.au's XML data feed made available in the data.gov.au catalogue and then converted into the Semantic Web format RDF using this code. That RDF data is then housed in an Apache Jena graph database made available on the web via the Fuseki URI which this API then queries as people or machines load particular web addresses.

The code for this API is available within the Australian Government Linked Data Working Group's GitHub account here. This API uses the pyLDAPI Python module created by CSIRO to enable Linked Data functionality for basci Python web applications.

Status of the tool

This API is only a small demonstrator. It will hopefully be replaced within the 2018/2019 Financial Year by a more robust tool that delivers RDF data for directory.gov.au data perhaps incorporated within directory.gov.au itself. The work to built this more robust tool is now being discussed by the Department of Finance (owners of directory.gov.au) and the CSIRO.

Persistent Identifier Demonstration

This API demonstrates persistent identifiers (PIDs) in action. Instead of using particularly human-readable web address to identify and find information out about something, say Geoscience Australia, which is https://www.directory.gov.au/portfolios/industry-innovation-and-science-part-jobs-and-innovation-portfolio/geoscience-australia in directory.gov.au and has https://www.ga.gov.au as it's own website, this API uses http://test.linked.data.gov.au/dataset/auorg/organisation/O-000887 where linked.data.gov.au is the Australian Government Linked Data Working Group's dedicated subdomain for persistent Linked Data resources.

Note that currently test.linked.data.gov.au rather than linked.data.gov.au is used here as this is a demonstration resource.

This persistent web address http://test.linked.data.gov.au/org/O-000887 uses a long-term stable key, O-000887, in addition to the dedicated web address for Australian Federal Government Organisations within this dataset, http://test.linked.data.gov.au/org/ to make a web address for each dataset member.

These persistent web address mechanisms owe much to governance arrangements made about the linked.data.gov.au namespace which is governed by the Australian Government Linked Data Working Group and a full explanation of the mechanisms under test here, when finalised by the Group, will be available on the Group's website linked.data.gov.au
